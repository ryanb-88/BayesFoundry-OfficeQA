## Failure Analysis

### uid0041 — Theil Index: Wrong Formula (NEW regression)
- **Question:** Theil index of dispersion for Rural Electrification Administration holdings FY1961-1970
- **Agent answer:** 0.005 | **Expected:** 0.011
- **Root cause:** Agent used the **normalized** Theil T formula: `T = (1/n) * Σ((xi/μ) * ln(xi/μ)) / ln(n)`, dividing by `ln(n)`. The standard Theil T index per Wikipedia is **unnormalized**: `T = (1/N) * Σ (xi/μ) * ln(xi/μ)`. The agent's unnormalized calculation (#5 in its output) was 0.011 — the correct answer — but it chose the normalized version.
- **Why it regressed:** The prompt's `theil_index()` function in the inlined compute.py uses shares (`v/total`) and computes `(1/n) * Σ(s * ln(n*s))`, which is the unnormalized Theil T. But the agent wrote its own implementation instead of using the provided function, and chose to normalize by `ln(n)`.

## Proposed Optimizations

### 1. Table Verification Rule (addresses Pattern 1 — 3 failures)
The agent needs to verify it found the RIGHT table, not just A table. Add a rule:
- After finding data, verify the **table name/number** matches what the question asks for
- For ESF questions: must find "Table ESF-1" or "ESF-1" in the section header
- For banking liabilities: must find "CM-I" or "Liabilities to Foreigners Reported by Banks" section
- For T-bill outstanding: must find "PDO-1" or "Public Debt Outstanding" summary table

### 2. Force Use of Provided Functions (addresses Pattern 2 — 1 failure)
The Theil index function in the prompt computes the correct (unnormalized) formula. Add a rule:
- "When the prompt provides a Python function for a calculation, USE IT. Do not write your own implementation."
- Fix the `theil_index()` function comment to explicitly say "unnormalized Theil T (standard formula)"

### 3. Cross-Reference Magnitude Check (addresses Pattern 1)
Add a sanity check table for expected value ranges:
- ESF Total Assets: $20-50 billion (20,000,000-50,000,000 in thousands)
- Total liabilities to foreigners (banks): $1-5 trillion range for 2003, $5-15T for 2013
- T-bill outstanding: $20,000-60,000 million

### 4. Model Upgrade Consideration
minimax-m2.5 is a mid-tier model. The persistent wrong-table failures suggest the model struggles with disambiguation when multiple similar tables exist. A stronger reasoning model (Claude, GPT-4o) might handle this better, though at higher cost.
